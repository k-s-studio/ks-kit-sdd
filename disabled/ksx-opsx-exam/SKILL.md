---

scope: This skill is used to examine the changes that have been implemented by agents. It should be used before committing the changes to ensure that all changes are correct and meet the requirements.

After `/opsx-apply` command done and some changes have been implemented, build it up and examine the changes to ensure that they are correct and meet the requirements.
- For Backend CRUD Implementation: Generate script tests to verify the correctness of the implementation. Ask User if you need an authorization token to access the API endpoints.
- For Client App Component Implementation: Generate e2e test cases to verify the correctness of the implementation. Open the web page to see how it actually works and check if it meets the requirements.
